# Go Microservices Example

This project contains three Go services that run together with Docker Compose:

- `api-gateway`: Gin API gateway with JWT validation and Redis user caching
- `user-service`: Gin user service with PostgreSQL, GORM, registration, and login
- `order-service`: Gin order service with PostgreSQL, GORM, and user-scoped orders

## Run

```bash
docker-compose up --build
```

Gateway is available at `http://localhost:8080`.

## Gateway Routes

- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/users/me`
- `POST /api/orders`
- `GET /api/orders`
- `DELETE /api/orders/:id`
- `GET /healthz`

## User Service Routes

- `POST /auth/register`
- `POST /auth/login`
- `GET /users/:id`
- `GET /internal/users/:id`
- `GET /healthz`

## Order Service Routes

- `POST /orders`
- `GET /orders`
- `DELETE /orders/:id`
- `GET /healthz`

## Example Requests

Register:

```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"alice@example.com","password":"password123"}'
```

Login:

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"alice@example.com","password":"password123"}'
```

Use the returned token:

```bash
export TOKEN="<jwt-from-login>"
```

Get current user:

```bash
curl http://localhost:8080/api/users/me \
  -H "Authorization: Bearer $TOKEN"
```

Create order:

```bash
curl -X POST http://localhost:8080/api/orders \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"product_name":"Mechanical Keyboard"}'
```

List orders:

```bash
curl http://localhost:8080/api/orders \
  -H "Authorization: Bearer $TOKEN"
```

Delete order:

```bash
curl -X DELETE http://localhost:8080/api/orders/1 \
  -H "Authorization: Bearer $TOKEN"
```

## Notes

- JWTs are generated by the user service and validated locally in the gateway.
- The gateway caches user JSON in Redis after the first successful lookup.
- Internal service calls use `X-Internal-Token`.
- User and order services run `AutoMigrate` on startup.
- Compose uses health checks for Redis and both PostgreSQL instances.
